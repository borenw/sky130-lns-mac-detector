# Two-Design Comparison — Spike Detector on SkyWater 130 nm (12-bit)

**Function (both designs):** `spike = (A·B + C·D) > Vth`, with `A,B,C,D` 12-bit
unsigned (0–4095) and `Vth` a 25-bit programmable threshold (covering the max sum
4095·4095·2 = 33,538,050). Both designs share the exact same 7-port interface
(`clk, A, B, C, D, Vth, spike`) and the same **2-cycle registered-I/O latency**
(74 flip-flops each: 4·12 + 25 input + 1 output).

| | Design 1 — baseline | Design 2 — log / LNS, K=1 |
|---|---|---|
| Method | exact `A*B + C*D`, one comparator | leading-one detector + 1 mantissa bit → log2; LNS add `max + F(\|Δ\|)`; compare in log domain |
| Multipliers | 2 × `$mul` | **none** |
| Golden spec | `spike_exact` (exact integer math) | `spike_k1` (the K=1 model — *this* is its "correct" answer) |
| Files | `rtl/mult_detector.v` | `rtl/log_detector.v` + `lod5.v`, `lns_ftable.v`, `lns_add.v` |

The K=1 helper bit is the single mantissa bit copied below the leading one — the
one bit that buys accuracy back over a pure leading-one-detector log. The
add-correction ROM `F(d) = round(2·log2(1+2^−d/2))` (47 entries, d=0…46) is
**generated by `model/model.py` and emitted as `rtl/lns_ftable.v`**, so the Python
model and the hardware ROM cannot drift.

## K=1 accuracy cost (Design 2 vs Design 1)

The 12-bit input space is 4096⁴ ≈ 2.8×10¹⁴ combinations — far too large to
enumerate — so the disagreement rate is a **Monte-Carlo estimate over 4,000,000
random samples × 12 Vth values** (48 M evaluations):

**Overall disagreement rate ≈ 5.60 %**

| Vth | disagree % | | Vth | disagree % |
|----:|-----------:|---|----:|-----------:|
| 0        | 0.000 | | 12,000,000 | 12.830 |
| 1        | 0.000 | | 16,769,025 | 5.304 |
| 1,000    | 0.000 | | 20,000,000 | 2.663 |
| 100,000  | 0.104 | | 25,000,000 | 0.353 |
| 1,000,000| 1.804 | | 33,538,050 | 0.000 |
| 4,000,000| 10.508| |  |  |
| 8,388,608| 33.629| |  |  |

Disagreement peaks in the mid dynamic range (Vth≈2²³), where many products land
near the threshold and the log quantization matters most; it vanishes at the
extremes. This is the *accuracy price of removing the multiplier*. A disagreement
is **not** an RTL bug — it is only a bug if the RTL output ≠ `spike_k1`, which is
exactly what Phase 4 checks. (Report: `report/model_accuracy.txt`.)

## Evidence: no multipliers, no latches, verified

- **No latches:** yosys elaboration reports "No latch inferred" for every signal
  in Design 2; `CHECK` pass finds 0 problems in both. (`report/elaboration.txt`)
- **No multipliers in Design 2:** 0 `$mul` cells (Design 1 has 2). The F(d)
  correction is a small ROM, exactly as intended.
- **Verification PASS** (`verif/sim_report.txt`): one testbench streams 63,875
  vectors (random + comparator-boundary + disagreement stress cases) through
  **both** designs, aligned to the 2-cycle latency:
  - Design 1 output == `exp_exact` → **0 mismatches, PASS** (bit-exact to exact math)
  - Design 2 output == `exp_k1`    → **0 mismatches, PASS** (bit-exact to the K=1 model)
- **Clean netlists:** both fully mapped to sky130 HD cells, **0 `$`-cells** left.

## Power + area (sky130 HD, `tt_025C_1v80`)

Area from `stat -liberty` (real cell areas). Energy is an **estimate**:
`E_op = α·(1+wire)·ΣCin·Vdd²` with input-pin caps read from the liberty —
assumptions `Vdd=1.8 V, α=0.15, wire=1.0×, f=50 MHz`. The robust figure is the
**×baseline ratio**, which is insensitive to those assumptions.

| Design | cells | area µm² | die x × y µm | energy/op pJ | power µW @50 MHz | ×baseline power | disagreement % |
|---|---:|---:|---:|---:|---:|---:|---:|
| mult baseline | 1476 | 12015.27 | 135.9 × 136.0 | 9.714 | 485.72 | 1.000 | 0.00 |
| **log K=1** | 416 | **3569.67** | **74.8 × 73.4** | **2.483** | **124.15** | **0.256** | 5.60 |

(`report/power_area.csv`, `report/floorplan.csv`)

## Takeaway

**Dropping the two 12-bit multipliers for a K=1 log/LNS detector cuts area by 70 %
(12015 → 3570 µm²) and estimated switching power by 74 % (0.26× baseline), at the
cost of a ≈5.6 % disagreement with the exact answer** — concentrated in the mid
dynamic range and zero at the extremes. The saving is much larger than at 5-bit
(−28 %) because multiplier area grows ~quadratically with width while the log
datapath grows ~linearly.
