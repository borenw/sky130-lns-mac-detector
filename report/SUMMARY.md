# Two-Design Comparison — Spike Detector on SkyWater 130 nm

**Function (both designs):** `spike = (A·B + C·D) > Vth`, with `A,B,C,D` 5-bit
unsigned (0–31) and `Vth` an 11-bit programmable threshold (0–2047, covering the
max sum 31·31·2 = 1922). Both designs share the exact same 7-port interface
(`clk, A, B, C, D, Vth, spike`) and the same **2-cycle registered-I/O latency**.

| | Design 1 — baseline | Design 2 — log / LNS, K=1 |
|---|---|---|
| Method | exact `A*B + C*D`, one comparator | leading-one detector + 1 mantissa bit → log2; LNS add `max + F(\|Δ\|)`; compare in log domain |
| Multipliers | 2 × `$mul` | **none** |
| Golden spec | `spike_exact` (exact integer math) | `spike_k1` (the K=1 model — *this* is its "correct" answer) |
| Files | `rtl/mult_detector.v` | `rtl/log_detector.v` + `lod5.v`, `lns_ftable.v`, `lns_add.v` |

The K=1 helper bit is the single mantissa bit copied below the leading one — the
one bit that buys accuracy back over a pure leading-one-detector log. The
add-correction ROM `F(d) = round(2·log2(1+2^−d/2))` is **generated by
`model/model.py` and emitted as `rtl/lns_ftable.v`**, so the Python model and the
hardware ROM cannot drift.  `F = [2,2,1,1,1,0,0,…]` (half-log2 units).

## K=1 accuracy cost (Design 2 vs Design 1)

Measured over the **full 32⁴ = 1,048,576 input space × 12 Vth values**
(12,582,912 evaluations):

**Overall disagreement rate = 621,489 / 12,582,912 = 4.94 %**

| Vth | disagree % | | Vth | disagree % |
|----:|-----------:|---|----:|-----------:|
| 0    | 0.000 | | 900  | 8.565 |
| 1    | 0.000 | | 961  | 5.939 |
| 31   | 0.138 | | 1000 | 4.565 |
| 100  | 4.989 | | 1500 | 0.273 |
| 256  | 21.897 | | 1922 | 0.000 |
| 500  | 12.905 | | 2000 | 0.000 |

Disagreement peaks in the mid-range (Vth≈256), where many products land near the
threshold and the log quantization matters most; it vanishes at the extremes.
This is the *accuracy price of removing the multiplier*. A disagreement is **not**
an RTL bug — it is only a bug if the RTL output ≠ `spike_k1`, which is exactly
what Phase 4 checks. (Report: `report/model_accuracy.txt`.)

## Evidence: no multipliers, no latches, verified

- **No latches:** yosys elaboration reports "No latch inferred" for every signal
  in Design 2; `CHECK` pass finds 0 problems in both. (`report/elaboration.txt`)
- **No multipliers in Design 2:** 0 `$mul` cells (Design 1 has 2). The F(d)
  correction is a small ROM, exactly as intended.
- **Verification PASS** (`verif/sim_report.txt`): one testbench streams 61,657
  vectors (random + comparator-boundary + disagreement stress cases) through
  **both** designs, aligned to the 2-cycle latency:
  - Design 1 output == `exp_exact` → **0 mismatches, PASS** (bit-exact to exact math)
  - Design 2 output == `exp_k1`    → **0 mismatches, PASS** (bit-exact to the K=1 model)
- **Clean netlists:** both fully mapped to sky130 HD cells, **0 `$`-cells** left.

## Power + area (sky130 HD, `tt_025C_1v80`)

Area from `stat -liberty` (real cell areas). Energy is an **estimate**:
`E_op = α·(1+wire)·ΣCin·Vdd²` with input-pin caps read from the liberty —
assumptions `Vdd=1.8 V, α=0.15, wire=1.0×, f=50 MHz`. The robust figure is the
**×baseline ratio**, which is insensitive to those assumptions.

| Design | cells | area µm² | energy/op pJ | power µW @50 MHz | ×baseline power | disagreement % |
|---|---:|---:|---:|---:|---:|---:|
| mult baseline | 264 | 2436.09 | 1.731 | 86.56 | 1.000 | 0.00 |
| **log K=1** | 211 | **1751.68** | **1.269** | **63.47** | **0.733** | 4.94 |

(`report/power_area.csv`)

## Takeaway

**Dropping the two multipliers for a K=1 log/LNS detector cuts area by 28 %
(2436 → 1752 µm²) and estimated switching power by 27 % (0.73× baseline), at the
cost of a 4.94 % disagreement with the exact answer** — concentrated near
mid-range thresholds and zero at the extremes.
